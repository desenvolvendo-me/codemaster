# PRD — IN005 Web

## Contexto

O domínio Web representa a interface web futura do CodeMaster — uma iniciativa pós-MVP sem requisitos funcionais definidos no momento. Esta iniciativa nascerá após a validação do método nas fases CLI (IN001), agente core (IN002) e integrações (IN003, IN004), quando houver evidência suficiente de que o ciclo de aprendizado funciona em uso real e exista demanda por visualização em equipe e acesso corporativo.

## Visão (Phase 2 — Growth / Phase 3 — Expansion)

**Phase 2 — Growth (pós-validação do método):**
- Versão web simples para visualização de evolução individual
- Dashboard para CTO/tech lead — legível por terceiros, sem configuração corporativa adicional
- Knowledge com sugestão de recursos específicos (artigos, projetos)
- Canvas automático do Obsidian para visualização de progresso

**Phase 3 — Expansion:**
- Marketplace de métodos (outros mentores publicam seus "20 anos")
- Sincronização de vault para visibilidade de time
- Portfólio de conhecimento navegável por empresa
- Relatório comparativo: "dev do Milestone 1 vs. hoje"

**Usuário secundário (Lucas — CTO):**
A jornada de Lucas documenta a necessidade de visibilidade do time: CTOs e tech leads precisam que seus times adotem IA com qualidade e consigam gerar valor real ao negócio. A versão web entrega esse acesso sem requerer que o time instale nada além do CLI atual — o Legend já é legível por terceiros no MVP, mas a web tornaria isso escalável para equipes inteiras.

**Meta de negócio associada:**
- 12 meses: Versão web lançada com primeiras empresas — ≥1 empresa adotando como processo interno

## Functional Requirements

Nenhum requisito funcional definido para esta iniciativa no MVP. Os FRs serão levantados após validação do método nas fases CLI e agentes, com base em feedback real dos usuários e das empresas que adotarem o CodeMaster.

## Non-Functional Requirements

Nenhum requisito não-funcional definido para esta iniciativa no MVP. Decisões arquiteturais dependem do modelo de acesso (SaaS, self-hosted, híbrido) e do modelo de dados (sincronização de vault, API centralizada, etc.) — a definir em fase futura.
